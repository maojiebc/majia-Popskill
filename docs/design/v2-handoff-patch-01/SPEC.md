# Popskill — 功能与逻辑规格 v2（第一性原理瘦身版）

> **可点击原型 = `prototype.html`**，为交互真源。本文供直接落地为真实应用。
> v1 规格（10 屏方案）留底于 `SPEC-v1.md`；本版按「不求功能多，只求满足场景需要」重新收敛。

---

## 1. 产品是什么（不变）

Popskill 是一个**本地 AI 能力管理器**。它在 `~/.agents/` 维护一份能力仓库（按类型分 `skills/ agents/ mcp/ bin/` 四个子目录），
通过 **symlink** 把每一项能力同时挂载到多个 AI 工具（Claude Code `~/.claude/`、Codex CLI `~/.codex/` 等）。

> **为什么是 `~/.agents/`**：`~/.agents/skills/` 是正在形成的跨工具共享约定（opencode、pi 等已原生扫描该目录）。
> 把 store 直接安在约定位置意味着：**原生读取 `~/.agents/skills/` 的工具零配置直接可用，symlink 只服务还认自家目录的工具**。

核心场景一句话：**技能装一次，挂到多个 AI 工具上；坏了能修，旧了能升。**

由此拆出的全部必要动作只有 5 个，整个产品围绕它们收敛为「一屏 + 两弹层」：

| # | 场景动作 | 承载 |
|---|---|---|
| ① | 看清我有什么、挂在哪 | 主屏（卡片矩阵） |
| ② | 装新的 | 「+ 添加」弹层（粘贴 URL → 安装计划） |
| ③ | 挂 / 摘某个工具 | 卡片上的工具 pill / 子项单元格，点击切换 |
| ④ | 修坏的、升旧的 | 行内徽标 + 顶部健康横幅（异常是例外态，不配独立页面） |
| ⑤ | 删 | 卡片悬停 ✕ / 设置中移除源 |

## 2. 数据模型

### 2.1 Entry（已添加的源）— 唯一顶层概念
每粘贴一个 URL 产生一条 Entry。**取消了 v1 的 Registry/AddedSource 双层结构**——
浏览发现不在产品内发生，因此只需记录「实际加过的 URL」这一层。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` / `name` / `desc` / `author` | string | 标识与展示 |
| `type` | enum | `Skill` / `Agent` / `MCP` / `CLI` / `Bundle` |
| `sourceUrl` | string | `github.com/owner/repo` · `npm:pkg` · `~/path`（kind 由 URL 形态推断） |
| `version` | semver | 当前安装版本 |
| `latest` | semver? | 上游最新版（更新检查写入；存在且 > version ⇒ 可更新） |
| `autoUpdate` | bool | 自动更新开关 |
| `tokens` | number | 提示词体量（CLI 为 0） |
| `children` | Capability[]? | 仅 Bundle 有；子项即 Capability |
| `claude` / `codex` | LinkStatus | 非 Bundle 时自身即能力，每工具一个状态 |

### 2.2 Capability（套装子项 / 独立能力本体）
`id` `name` `type` `desc` `version` `author` `tokens` + 每工具一个 LinkStatus + 可选 `brokenCause`。
另有 `readme`（SKILL.md / README 开头摘要，安装时从源文档提取，原型中为静态样例 `psReadmes`），供详情 peek 展示。

### 2.3 LinkStatus
| 值 | 含义 | 视觉 |
|---|---|---|
| `on` | symlink 有效 | 蓝 ● |
| `off` | 未链接 | 灰 ○/— |
| `stub` | 占位待校验 | 琥珀 ◐ |
| `broken` | 目标丢失/校验失败 | 红 ✕ |

`brokenCause`: `断链`（目标丢失）/ `源失联`（上游 404）/ `校验失败`（manifest/签名不过）。

### 2.4 Tool
`id` `name` `root` `connected` `defaultTarget`。原型画 Claude/Codex 两列；真实支持 N 个工具。

## 3. 信息架构

**没有侧栏、没有路由。** 一个主屏 + 两个覆盖层 + 派生的空态：

- **主屏：能力矩阵（卡片式）** — 独立能力为双列卡片，套装为通栏卡片
- **添加弹层** — 标题区「+ 添加」/ 空态 CTA 触发
- **设置弹层** — 标题栏 ⚙ 触发
- **空 store 态** — entries 为空时主屏自动切换

标题栏：品牌 · 同步指示 chip · ⚙。状态栏：store 路径 + 「↗ 在编辑器中打开」 · symlink 数 · 断链/占位计数 · 同步时间 · 版本。

---

## 4. 主屏：卡片矩阵

视觉：紧凑账本配色（米白纸面 `#fafaf8/#f4f2ec`、发丝线 `#e8e6df`、单一电光蓝 `#1f4ed8`，
琥珀/红仅用于 stub/broken），卡片布局取自早期 02 方向但去掉浓郁色彩（无彩色头像、无色带）。

- **独立能力卡**（双列网格）：中性单字头像 + 名称 + 类型 tag + 描述 + 元信息（版本/作者/tokens/源 URL），右列两枚工具 pill（Claude / Codex），pill 显示状态文案（已激活/未链接/占位/断链）。
- **套装卡**（通栏）：头部 = 折叠箭头 + 名称 + Bundle tag + 描述 + 源 URL + 两侧聚合分数 `5/8` + 迷你覆盖条 + 版本；展开后内部是紧凑子项清单（树形缩进 + 类型 tag + 两列状态单元格 + 版本）。
- **搜索**（快捷键 `/`）：按名称/描述/作者过滤，命中高亮；命中子项时自动展开所属套装。
- **类型 chip 行**：全部 / Skill / Agent / MCP / CLI / Bundle。非「全部」时子项摊平为独立卡（标注 `⊂ 所属套装`）。
- **悬停操作**：↗ 在编辑器中打开（能力本质是文件，编辑去编辑器——这替代了 v1 的应用内编辑器）；✕ 移除（仅顶层条目）。
- **详情 peek**：点击任意能力名称（独立卡 / 套装子项，悬停下划线提示）弹出 380px 小浮层：完整描述 → 「SKILL.md · 文档摘要」引文块（`readme` 字段）→ 两侧链接状态（状态色 + 文案）→ 来源 URL；头部 = 名称 + 类型 tag + 版本/作者/tokens/所属套装；底部 = 「↗ 在编辑器中打开」+ 「完整文档在 SKILL.md」。锚定点击位置，下半屏自动上翻，Esc/遮罩关闭，与修复弹层互斥。设计意图：一行描述不够时的「这个 skill 是干什么的」速查，仍不做整页详情。

### 4.1 行内健康处理（替代 v1 修复中心 + 更新中心）

- **健康横幅**：有 broken 或可更新时出现在标题区下方：`✕ N 个链接问题 · ↑ M 个源可更新` + 「全部修复 (N)」「全部更新 (M)」。归零即消失。
- **✕ / ◐ 点击 → 修复弹层**（锚定单元格，下半屏自动向上翻转）：标题 = 成因 + 能力 + 工具侧；来源行（URL + 可更新标记）；2-3 个方案，推荐项标绿：
  - 有新版 → **更新到 x.y.z 并修复**（推荐）
  - 断链 → 重链到 store 本地版本（推荐）/ 从源重新拉取 / 移除该侧链接
  - 校验失败 → 从源重新拉取并校验（推荐）/ 跳过校验启用（降级）/ 移除该侧链接
  - 占位 → 完成校验并启用（推荐）/ 移除该侧占位
- **版本 ↑ 徽标**：套装头/卡片元信息/设置源列表中均可点击，一步更新（拉新版 → 重链，保留各工具链接关系，顺带修复该源下全部 broken）。
- 「全部修复」对每个问题执行其推荐方案。

### 4.2 ⌘K / 批量操作 / 图谱
已砍。搜索由主屏搜索框承担；批量场景由「全部修复/全部更新」与源级移除覆盖。

---

## 5. 添加弹层（替代 v1 源/更新中心）

两步：
1. **粘贴 URL**（GitHub 仓库 / npm 包 / 本地路径，示例 chip 可一键填入）→ 解析。
2. **安装计划**：来源卡（kind tag + URL + 版本）→ 提供的能力清单 → 挂载目标开关（默认值来自 Tool.defaultTarget）→ 「将写入」终端预览（store 路径 + ln -s 命令）。
   主按钮：`安装并链接 (N)`，无目标时 `仅保存到 store`。

完成后新条目置顶并高亮闪烁。**浏览/发现（GitHub stars、ClawHub、npm 热度 tab）整体移除**——发现新技能发生在产品外。

---

## 6. 设置弹层（吸收源管理）

四个分区，一页滚动：
1. **已添加的源**：每条 Entry 一行 = kind tag + URL + 提供什么 + 版本（+↑ 可更新徽标，点击即更新）+ 自动更新开关 + 移除（联动卸载能力与 symlink）。
2. **工具**：连接状态点 + 名称 + 根目录 + 「新安装默认挂载」开关；+ 添加工具。
3. **Store 与同步**：路径 + 在编辑器中打开；同步后端（iCloud Drive）。store 跨设备同步，symlink 为各机本地态。
4. **关于**：版本 / store 体积。

---

## 7. 空 store 态（首次启动）

主屏自动切换为：`~/.agents — 空` + 「+ 粘贴 URL 添加」（开添加弹层）+ 「扫描本地目录」。

---

## 8. 关键逻辑（不变项摘自 v1）

1. **链接解析**：`readlink ~/.<tool>/<kind>/<name>` → `~/.agents/<kind>/<name>-<version>/`；目标不存在 = broken；校验失败 = broken(校验失败)；未链接 = off。
2. **安装**：fetch 到 store（校验签名）→ 对选中 Tool 建 symlink；套装对每个子项重复。
3. **更新**：比对上游版本写 `latest`；更新 = 拉新版 → 重指 symlink（保留各工具链接关系）。
4. **修复**：全部落到 relink / repull / update / trust / unlink / remove 之一（见 §4.1）。
5. **同步**：store 经 iCloud Drive/Git 同步；symlink 不同步。

## 9. 本轮裁剪（相对 v1 的 10 屏方案）

| 移除项 | 去向 / 理由 |
|---|---|
| 侧栏 + 路由 | 只剩一个目的地，侧栏失去存在意义 |
| 修复中心独立页 | → 行内修复弹层 + 健康横幅（异常是例外态） |
| 源/更新中心（含浏览发现 tab） | → 添加弹层 + 行内 ↑ 徽标；发现新技能发生在产品外 |
| Registry/AddedSource 双层源模型 | → 单层 Entry；浏览没了，「注册表连接」随之失去意义 |
| 新建能力（应用内 Markdown 编辑器） | → 「↗ 在编辑器中打开」；能力是文件，编辑属于编辑器 |
| 组装套装 | 发布者工作流，非管理器场景 |
| ⌘K 命令面板 | 主屏搜索已覆盖 |
| 批量操作页/多选态 | 「全部修复/全部更新」+ 源级移除覆盖真实批量场景 |
| 统计条（本月 Tokens 等） | 数字装饰；副标题一行计数足够 |
| 调用次数列 | 同上；保留 tokens（关乎提示词预算） |

## 10. 待定 / 假设

1. **多工具**：原型两列；真实 N 列（每工具一个 LinkStatus、一枚 pill）。
2. **trust（跳过校验）**：降级操作，是否保留需确认风险提示方案。
3. **本地源监视**：手动收录 vs watch 自动，需确认。
4. **解析器**：原型对任意 URL 返回单项 Skill 假数据；真实需读 manifest（可能多项 → 计划清单已支持）。
5. **数据**：原型样例 = 3 套装 + 15 独立能力（`popskill-data.jsx`）；真实接 CLI / 文件系统。
6. **readme 摘要提取策略**：建议取 SKILL.md frontmatter 的 description + 正文首段（限 ~120 字）；CLI 无 SKILL.md 时取 README 首段或 `--help` 首行；安装/更新时提取并缓存，不在打开 peek 时实时读文件。
