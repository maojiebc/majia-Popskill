# Popskill v0.1 Release Readiness

Last checked: 2026-05-13

Popskill v0.1 is in release-candidate shape for local development builds. The remaining blockers are external release credentials and public update metadata, not product implementation work.

## Dry Run Result

```bash
./scripts/ci-local.sh
```

Status: passed.

Covered by the dry run:

- Shell script syntax checks.
- Rust sidecar build and 38 tests.
- Read-only sidecar smoke, including WebDAV sync plan and Agent commands.
- SwiftUI app build and 52 tests.
- Native app launch smoke.
- Bundled `.app` launch smoke.
- Screenshot asset smoke, including dimensions, size, and low-variance checks when Pillow is available.
- Release artifact smoke for DMG, manifest, and appcast generation. The smoke appcast is written to a temporary path so placeholder URLs do not overwrite `build/appcast.xml`.

```bash
./scripts/release-doctor.sh
```

Status: expected not-ready before Apple Developer credentials.

Current checks that pass:

- Required tools: `codesign`, `security`, `ditto`, `hdiutil`, `install_name_tool`, `shasum`, `jq`, `otool`, `notarytool`, `stapler`.
- `build/Popskill.app` exists.
- `build/Popskill.dmg` exists.
- App executable exists.
- App executable links `Sparkle.framework`.
- Bundled Sparkle framework exists.
- App executable has `@executable_path/../Frameworks` rpath.
- `release-doctor` can compare app bundle version and bundle identifier against `POPSKILL_APP_VERSION` and `POPSKILL_BUNDLE_IDENTIFIER`.

Current expected blockers:

- `POPSKILL_DEVELOPER_ID_APPLICATION` is not set, and no Developer ID Application identity is currently available in Keychain.
- Notary credentials are not configured through `POPSKILL_NOTARY_KEYCHAIN_PROFILE` or the Apple ID environment variables.
- Public Sparkle feed URL, public EdDSA key, download URL, and update signature are not configured yet.
- `build/appcast.xml` is absent until a real public download URL/signature is provided. `generate-appcast.sh` rejects `example.com` placeholder URLs outside smoke tests, and `release-doctor` rejects placeholder URLs in an existing appcast.

## Release Gate

Before sharing with external testers:

1. Install a Developer ID Application certificate.
2. Store notary credentials with `xcrun notarytool store-credentials`.
3. Generate or fetch the Sparkle public EdDSA key.
4. Build and notarize with `./scripts/notarize.sh`.
5. Run `./scripts/package-dmg.sh`, `./scripts/release-manifest.sh`, and `./scripts/generate-appcast.sh` with final `POPSKILL_APP_VERSION`, `POPSKILL_APP_BUILD`, and `POPSKILL_BUNDLE_IDENTIFIER` values.
6. Verify Gatekeeper with `codesign`, `stapler`, and `spctl`.

See [release-runbook.md](./release-runbook.md) for exact commands.
