# Popskill v0.1 QA Checklist

Use this checklist after `./scripts/ci-local.sh` passes and before sharing a DMG with an external tester.

## Automated Gate

```bash
./scripts/ci-local.sh
./scripts/release-doctor.sh
```

Expected before Developer ID credentials are configured:

- `ci-local` passes.
- `release-doctor` confirms app bundle, DMG, release manifest, Sparkle framework/rpath, and configured Sparkle metadata are internally consistent.
- Before Developer ID credentials are configured, `release-doctor` fails only on Developer ID / notary credential readiness and Sparkle public feed/signature warnings.
- Public Sparkle-enabled release checks should run with `POPSKILL_REQUIRE_SPARKLE=true` so missing feed/key/download/signature/appcast metadata are failures.
- Screenshot smoke verifies required PNGs, dimensions, byte size, and low-variance/blank-image risk when Pillow is available.

## Visual Pass

Use a 1400 x 988-ish window first, then a narrower window around 1000 px wide.

- Sidebar sections are readable, clickable, and badges do not crowd labels.
- Discover rows keep `Plan` / `Install` actions aligned and readable.
- Library rows keep title, description, AgentShield badge, and Claude/Codex/Gemini toggles from overlapping.
- Agents page handles an empty `~/.claude/agents` state cleanly.
- Agents detail pane remains readable when target diagnostics are present.
- Repositories, Updates, Backups, and Settings do not show disabled primary actions as the dominant element.
- Usage, Token Spend, and Idle Candidates stay legible with real transcript data.
- Empty states do not include instructional marketing copy or oversized hero text.
- Text does not clip inside buttons, pills, rows, or detail fields.

## Data And Privacy Pass

- Insights must aggregate transcript metadata without displaying message body text.
- WebDAV status must not display a password.
- WebDAV configuration must pass a new password through `POPSKILL_WEBDAV_PASSWORD`, not argv.
- WebDAV sync plan must remain read-only and report the CC Switch boundary instead of uploading or downloading.
- Agent catalog failures from GitHub rate limiting must suggest `GITHUB_TOKEN` or `GH_TOKEN`.
- Agent install plans must remain read-only and must not create agent files.

## Release Artifact Pass

- `build/Popskill.app` launches with the bundled `skill-cli` when `POPSKILL_CLI` is unset.
- `build/Popskill.app` contains `Sparkle.framework`, and the app executable has an `@executable_path/../Frameworks` rpath.
- `build/Popskill.dmg` mounts and contains `Popskill.app` plus an `Applications` symlink.
- `build/release-manifest.json` contains version, build, artifact name, sha256, bytes, and generated timestamp, and `release-doctor` confirms those values still match the current app bundle and DMG.
- `build/appcast.xml` contains the Sparkle namespace, version fields, minimum system version, download URL, and public update signature when signing env is configured.
- GitHub Release starts as a draft with `Popskill.dmg`, `release-manifest.json`, and `appcast.xml`; mark it latest only after the uploaded DMG passes Gatekeeper from a clean macOS user account.

## Notarized Build Pass

Run this only after Apple Developer ID credentials are configured:

```bash
./scripts/notarize.sh
./scripts/package-dmg.sh
codesign --verify --deep --strict --verbose=2 build/Popskill.app
xcrun stapler validate build/Popskill.app
spctl --assess --type execute --verbose=4 build/Popskill.app
```

Finally, copy the app from the DMG into `/Applications` on a clean macOS user account and open it once.
