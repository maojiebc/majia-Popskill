## 为什么单独有这份文档

主 playbook 已经定了 GitHub Release / Product Hunt / HelloGitHub / V2EX 这些**广播渠道**。但 v0.1 真正的胜负不在广播流量，**在头部 Skills 开发者愿不愿意试一下并讲一句**。

2026-05-11 dotey 那条 126K views 的长推、2026-05-13 op7418 那条接力推，已经把"头部 Skills 开发者"群体名挑明了——「藏师傅、宝玉、乔木、一泽」四个人。他们当周在公开喊「需要一个聚焦的公共空间去分发、展示、曝光、连接，并服务开发者」（op7418 评论 #2 原话）。

popskill v0.1 就是这个东西的一个具体回答。**不接触这四个人就是浪费现成的语境**。这份文档就是为了让接触不沦为"撒名片"，而是基于他们自己说过的话、做过的事，给出有诚意的接洽。

**为什么必须接触**：

## 全局红线（适用所有 Tier）

1. **不付费** —— 任何形式的现金、装备、推广预算
2. **不带货语气** —— 不写"求支持""求一键三连""帮个忙转推"
3. **不强求公开评价** —— 私信后无回复就 stop，不重复发送
4. **不假装认识** —— 没合作过就是没合作过，话术里不暗示"我们之前聊过"
5. **不在他们 timeline 公开塞链接** —— popskill 链接只出现在自己的 tweet、自己的 README、自己的 Release page
6. **保护他们的注意力** —— 一次私信用最少字数把要点说清，不附超过 1 张截图

## 失败兜底

如果 Tier 1 两位都不回复 + 发布日 Tier 2 也没人 quote / reply：

- **popskill v0.1 依然按原计划发布**，不延期
- **不在公开渠道暗示"被忽视了"或"想要他们回应"**
- 走 v0.1-launch-playbook.md 的 T+7 HelloGitHub / awesome list 流程
- v0.2 周期再找时机自然接触（比如他们再发相关内容时点赞 + 简短评论）

KOL outreach 是 **加分项**，不是发布门槛。
