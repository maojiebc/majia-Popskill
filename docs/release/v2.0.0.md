# Popskill v2.0.0 — 第一性原理重写

从头重写的大版本。回到最初的问题：**技能装一次，挂到多个 AI 工具上；坏了能修，旧了能升。**

## 变了什么

- **一个屏幕**：能力矩阵（卡片式）+ 添加/设置两个弹层 + 行内修复弹层 + 详情 peek。没有侧栏、没有路由。
- **文件系统就是数据库**：store 在 `~/.agents/`（正在形成的跨工具约定目录），工具侧（`~/.claude` / `~/.codex`）只是 symlink。无 sidecar、无 SQLite——你的目录结构本身即全部状态。
- **套装（Bundle）**：含多个 skill 的目录自动识别为套装，支持整套挂载或逐子项开关（自动物化）。
- **修复**：断链 / 本地副本占位一键处理，危险操作全部走 store 回收站（`~/.agents/.trash/`），可恢复。
- **详情 peek**：点能力名称看 SKILL.md 摘要，深读走「在编辑器中打开」。
- **添加**：粘贴 GitHub 仓库或本地路径 → 安装计划（含 `ln -s` 预览）→ 一键安装并链接。

## 相比 v1.x

- 删除 Rust sidecar 与 cc-switch 依赖，17.6k 行 Swift + 4.8k 行 Rust → ~3k 行纯 Swift
- SSOT 从 `~/.cc-switch/skills` 迁移到 `~/.agents/`（已有该拓扑的用户零迁移）
- v1 的 Discover / Insights / Compose / WebDAV 等已按「不求功能多，只求满足场景」裁剪
