# v2.14.0 — 更新体验大修 + CLI 巡检 + 双语热门目录

围绕「更新」这条主线的一次系统性补课，外加精选目录的双语化与热门扩容。

## 更新体验四连修

- **计数按技能数**：横幅与「全部更新」按钮的数字改为待更新技能总数——套装里 5 个成员有新版就显示 5，不再按源数报 1（`Entry.updateCount`）。
- **提示可点击跳转**：横幅「N 个技能可更新」现在是按钮——点击滚动定位到待更新条目，套装自动展开、闪烁高亮，多个时循环跳；被搜索词/类型筛选挡住时自动清过滤。
- **主界面常驻「检查更新」按钮**：hero 右侧新增（曾只埋在设置弹层里），检查中转 spinner，结果 toast 如实报告。
- **表格视图补更新徽标**：v2.13 引入表格视图时漏了整套更新提示——套装头行、独立行、子项行全部补齐（子项迷你角标 ↑ 点击即更新所属源）。

## npm 源与全局 CLI 进更新雷达

- **npm 源真检查**（曾整体跳过）：以 @guandata/guanskill 为标本——npm 包发布的是 CLI 本体，技能目录是 `install-skill` 生成的，所以更新语义 = registry 最新版 vs `npm -g` 已装版，更新 = `npm i -g`，绝不碰 store 技能目录。
- **CLI 巡检面板**（吸收 cc-switch 工具版本矩阵）：`npm ls -g` 全局包逐个对比 registry——claude-code / lark-cli / getnote 们从此有版本矩阵和一键升级。入口：横幅「⌨ N 个 CLI 可升级」+ 设置页 + 应用菜单。npm 探测走 login shell（GUI app 的 PATH 没有 ~/.local）；未装 Node 的机器整块静默隐藏。

## well-known 源全链支持

skills.sh 生态 2026-06 起把 lark 系 24 个技能的 lock 记录改写成 `open.feishu.cn/.well-known/skills/<名>/SKILL.md` 单文件分发地址。此前这类源被 normalizeSource 截成 `.well-known/skills` 当 github 源——套装名显示成 `.well-known/skills`，更新检查去 clone 一个不存在的仓库必失败。

- 归拢键 = `wk:<域名>`，lark 系归成一张干净的「open.feishu.cn」卡（实测 27 项）。
- 更新检查 = 逐成员 GET SKILL.md 比内容哈希；更新 = 原子换 SKILL.md（旧文件进回收站），references/ 附属文件保留。
- 「添加」框粘 well-known 地址可直接安装。
- 已知局限（协议使然）：references/ 等附属文件不在单文件协议内，其变化检不出。
