# v2.16.0 — 存量功能大扫除

不加新功能的一版：四视角审计现有能力（更新链 / 主界面 / 弹层流程 / 引擎数据），40 条发现修掉 30 条。主题一句话：**做了的决定看得见，做完的动作有收工，坏了的数据不静默。**

## 三个真 bug（升级即生效）

- **源式套装的「自动更新」从来没真正生效过**：设置页开关写 meta 用的键与扫描回读的键不同，开完任何一次刷新就悄悄回落。baoyu / lark 这类主力套装若你开过自动更新——这版之后才是真的开了。
- **「停用」的定时任务重启电脑会自己复活**：旧实现 `launchctl unload` 不写 override，plist 留在 LaunchAgents 下次登录自动重载。现在 `bootout + disable` 真停用（plist 仍原样保留）。
- **展开套装后键盘顺序错乱**：展开的套装被提升到网格最前，↑↓ 焦点却按提升前的顺序走。

## 数据完整性（引擎层）

- **meta 孤儿清理**：终端删掉的技能残留的 sourceUrl / autoUpdate 记录会祸害重装的同名技能（最坏被旧仓库内容静默覆写）——refresh 时后台 GC。
- **meta 损坏不再静默归零**：`.popskill.json` 损坏（如 store 走 git 同步撞出冲突标记）时先备份 `.corrupt` 留第一现场并告警，不再让下一次写操作拿空表覆写。
- **symlink 指对才算「on」**：指到别处旧副本的链接曾被谎报健康挂载，现判 broken 并注明「指向别处」；相对链接与 store 内软链成员照常兼容（真实环境 156 条链接零误判验证）。
- **物化套装全开自动收敛回整链**：曾永久物化——上游新增子项默认漏挂；现在全 on 且目录内容可证明安全时换回整套一条 symlink，有任何真实文件保持物化绝不删。
- **回收站**：同秒撞名条目不再解析出脏名/丢时间；新增「清空回收站」按钮（带确认）；设置页回收站区随移除/导入实时刷新，不再显示旧清单。

## 反馈如实（有头必有尾）

- 「全部更新」「全部升级 CLI」收工出聚合账单（N 成 M 败点名），不再只报开工。
- npm CLI 升级排成串行队列——并发 `npm i -g` 会互咬全局目录；成功文案不再谎称「旧版已入回收站」（npm 升级根本不走回收站）。
- 恢复更新提醒时若全量检查在跑，排队等收尾后自动复查——曾 toast 说「正在重新检查」实际被静默吞掉。
- well-known 源部分成员没查成时如实透出数量；多成员更新中途失败报「已换 N 项后失败」。
- 导入未托管分账：「导入 4 个；2 个因同名跳过」；中途失败留下的半拷贝自动清理（曾永久卡死重试）。

## 跳过态可见 + 交互补全

- v2.15 的「跳过此版本」有了全套回显：卡片 / 表格 / 详情 peek / 设置页统一灰徽标「已跳过」，点击即恢复提醒——不再消失在只能右键试探的黑洞里。
- 修复弹层支持键盘：↑↓ 换方案、回车执行、数字 1-4 直选（空格开弹层后不再只剩 Esc）。
- Esc 一层层退：输入框编辑中第一下只退出编辑，不再连备注草稿带整个面板一起吞。
- 套装头行右键新增「查看详情」；卡片子行补右键菜单；表格行与展开套装头行补 VoiceOver 动作兜底。
- 添加流程：大仓安装移到后台（不再卡死 UI）+ 安装失败驻留错误条 + 同名冲突给出路 + 未装工具挂载行标注并复用确认（曾绕过防线静默建出 ~/.codex）+ 解析中退出不再泄漏临时 clone。
- 粘 npmjs.com 包页 URL 正确识别为 npm 源并给引导（曾报「无法识别 GitHub 仓库」）。
- CLI 巡检面板打开即后台重扫（曾长期显示启动时的旧版本号）；定时任务倒计时每 30 秒走表（曾冻结在打开那一刻）。
