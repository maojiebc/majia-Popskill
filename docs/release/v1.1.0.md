# Popskill v1.1.0 — 紧凑账本 redesign

A ground-up UI redesign. The whole app moves to the "紧凑账本" (compact ledger)
language — a warm-paper, light-only aesthetic with a single electric-blue accent
— and gains the screens the design prototype called for.

## What changed

### A new design language
- Global warm-paper palette (fixed light tokens), forced light appearance, and a
  single electric-blue accent throughout.
- Custom ledger window chrome: a titlebar with the symlink brand mark, a fixed
  text-only sidebar, and a foot status bar.

### Matrix, rebuilt
- Dense, edge-to-edge ledger table: 名称 / 类型 / 作者 / Claude / Codex / 版本 /
  Tokens / 调用.
- Per-tool link state is now a tappable `●/—/◐/✕` glyph; colored type tags;
  bundle rows show a coverage fraction + mini-bar with tree-connector children.
- Three-band header (hero · stats · filters) with a single type-chip row.

### Full-page Inspector
- Clicking a row opens a full-page detail (was a cramped side panel): breadcrumb
  + hero + coverage cards + component list + a source / SSOT / version /
  this-machine rail.

### New & reworked screens
- **新建能力** (Create) and **组装套装** (Compose) — author a capability or
  assemble a bundle, with live preview / coverage / manifest.
- **修复中心** (Fix) — health + repair, with an all-clear state.
- **源 / 更新** (Sources) — updatable items + your GitHub sources + URL-add.
- **设置** (Settings) — tabbed (连接 / 同步 / 源 / 安装 / 配额 / 关于).

### Information architecture
- The sidebar is now the 6 prototype destinations (矩阵 / 修复中心 / 源·更新 /
  新建能力 / 组装套装 / 设置). Earlier maintenance-only screens (insights /
  backups / idle) folded away; usage now lives in the matrix + inspector.

## Tests

- `swift test` = 150/150
- Rust sidecar untouched

## Migration

v1.0.5 → v1.1.0: Sparkle handles it. Start Popskill, accept the update prompt,
