# Popskill v1.0.4 — surface unification + selection fixes + Insights streaming

Follow-up to v1.0.3. v1.0.3 用户会自动收到升级提示。

## What changed

### Bug fixes

- **Spotlight / Idle / Link Health 跳转修复** — 之前从 ⌘K Spotlight 选一条 skill、从 Idle view 点 "Inspect"、从 Link Health 点行进矩阵，inspector 经常打不开正确的 skill。根因是 raw skill id 和 namespaced matrix capability id (`skill:xxx`) 没分清楚。现在两边各走各的，pending toggle 也用同一套 namespaced key 避免冲突。
- **Onboarding SSOT 文案再修** — wizard 步骤 2 的"扫描能力"说明仍提到 `~/.agents/skills`（一个 v0.3 时代的过时引用），实际扫描的是 `~/.cc-switch/skills`。两处都改成正确的。

### UX hardening

- **数据源删除前 confirmation dialog** — 之前 Sources view 右键 → 删除是即时删除，无确认。现在弹"确定删 owner/name?"对话框，避免误触。
- **Loading guards + 同步状态** — 各 view 的 refresh / update / sync 按钮在请求进行中正确 disabled。同步 provider 未实现的（WebDAV / iCloud SOON 标签的）按钮真正灰掉，不再表面可点击。
- **Update all 安全化** — 扫描和单条更新进行中不可再次点击"全部更新"，避免堆栈并发请求。

### Visual refinement

- **设计 token 统一** — v1.0.3 加的 canvas glow blur 圆球撤了（视觉收益有限 + GPU 成本），canvas / card / overlay surface 全部走 Popskill token 体系。后续 dark mode 调色集中到一个文件。
- **Accessibility / help copy 本地化** — 一批之前硬编码的 toggle label / hint text 改走 LocalizedText。

### Performance

- **Insights transcript scan 重写成 streaming** — 之前 `TranscriptUsageScanner.scan(fileURL:)` 一次性读整个 `.jsonl` 文件进内存再 split lines。新版按行 streaming，内存占用从 O(file size) 降到 O(line size)。majia 自己机器上 223 个 jsonl 文件总计几百 MB,扫描内存峰值从 ~600MB 降到 ~50MB。
- **Hot UI path 减少重复计算** — Matrix / Inspector 里 ByteCountFormatter / DateFormatter 等重复实例化挪出每帧路径。
