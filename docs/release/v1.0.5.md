# Popskill v1.0.5 — package matrix + inspector evolution

This release folds the recent HTML-reference UI iterations into one stable build. v1.0.4 users should receive the upgrade through Sparkle.

## What changed

### Package bundles become first-class

- **Suite rows in the matrix** — composite packages now appear as first-class matrix rows, with component expansion, Claude/Codex coverage, version identity, usage metrics, and broken-link aggregation.
- **Package inspector** — bundle details now include Overview, README, Usage, Version, Sync, and Metadata tabs, plus component state badges, component versions, usage counts, local paths, and safe package actions.
- **Coverage explanations** — bundle coverage cards now show ratio, percent, and stub/off gap labels, so partial coverage explains itself instead of stopping at `2/3`.

### Faster scanning and keyboard workflows

- **Usage metrics everywhere they matter** — matrix rows, single-skill inspectors, bundle inspectors, and expanded components now show 30-day calls, tokens, and trend context from the cached transcript usage index.
- **Command-K / Spotlight upgrades** — bundles can be found by package name, source, component name, component kind, status, and CJK trigger variants. Empty-query results are ranked by recent 30-day usage before lifecycle recency.
- **One-command refresh** — Spotlight now includes an action to re-scan usage and jump to Insights.

### Search, filters, and diagnostics

- **Broader matrix search** — search now covers source/author, enabled app targets, trigger phrases, package source, component kind/status, config fields, and normalized compact/spaced aliases such as `baoyu-comic` / `baoyu comic`.
- **Sidebar shortcuts** — Matrix sidebar shortcuts expose suites, Claude-only, Codex-only, broken links, inactive capabilities, skills, CLI tools, MCP servers, and agents with cached counts.
- **Sync and dependency diagnostics** — Settings and inspector surfaces now show structured sync status, last-sync details, link health, required command availability, and update state.

### UI fit and stability

- **HTML-aligned matrix density** — the first screen now has tighter KPI cards, compact filter chips, lighter table chrome, better bundle/component hierarchy, and a refreshed matrix screenshot.
- **Inspector overview upgrades** — skill and package overview pages surface this-machine usage, source/path metadata, companion package context, and safe local actions without requiring tab switching.
- **CI hardening** — PR checks now avoid duplicate branch builds, select Xcode 16 for Swift Testing, add step timeouts, and keep SwiftUI store-backed views MainActor-safe.
