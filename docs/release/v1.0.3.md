# Popskill v1.0.3 — UI polish + update lookup performance

UX polish release. Matrix view gets named design tokens, hover/selection
feedback, a real "no results" state, and an O(1) update-pending lookup that
was previously O(n²) when filtering by "可更新". v1.0.2 users will be
prompted by Sparkle automatically.

## What changed

### Visual polish

- **Hover state** for matrix rows — soft accent fill when you mouse over a
  row, with `prefers-reduced-motion` respected.
- **3pt accent indicator bar** on the leading edge of the selected row.
- **Search field focus state** — accent-colored border when the field is
  active; clear button (×) appears when text is present.
- **Filter chip borders** — every chip now has a hairline stroke so active
  and inactive states read clearer in dark mode.
- **Canvas background depth** — two softly-blurred accent circles add
  ambient lighting without competing with content.
- **Card surfaces** consolidated to named design tokens (`popSurfaceHover`,
  `popSelectedRowFill`, `popControlFill`, `popCardStroke`,
  `popAccentSoft`, etc.) — future dark-mode tweaks no longer require
  hunting through `Color.black.opacity(...)` literals.

### Usability

- **No-results state** — when filters or search narrow the matrix to zero
  rows, you now see a dedicated "nothing matches" screen with a one-click
  reset, instead of the global "no capabilities yet" message that
  previously misled people into thinking their data was gone.
- **`accessibilityHint`** on every matrix row pipes the capability summary
  into VoiceOver.

### Performance

- `PopskillStore.updates.didSet` rebuilds an `updateSkillIDs: Set<String>`
  cache. `MatrixRow.hasUpdate` and `MatrixFilter.includes(.updates)` now
  look up in O(1) instead of scanning the full updates array on every row
  redraw. With 60+ rows and a populated update list, the old code was
  effectively O(n²) on every matrix re-layout.

## Migration

v1.0.2 → v1.0.3: Sparkle handles it. Open Popskill, accept the upgrade
prompt, restart. No data changes.

Earlier versions: see [v1.0.2 release notes](./v1.0.2.md) for the SSOT
path fix and global error toast.
