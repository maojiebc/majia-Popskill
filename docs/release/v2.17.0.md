# v2.17.0 — 上游新增一键装 + 深链接

v2.16 大扫除时裁掉记入 roadmap 的欠账，这版全部还清：monorepo 上游新出的技能从「只在 toast 尾巴闪 2.6 秒」变成可见、可点、可批量安装；外部还多了一条 `popskill://` 直达通道。

## 上游新增技能：看得见，一键装

你装了 baoyu / lark 这类 monorepo 套装的一部分，上游每月都在加新技能——此前检查更新只在更新完成的 toast 里提一句「上游另有 N 个未安装技能」，看不到是哪几个、也没有任何后续动作入口。

- 检查更新现在把**上游新增名单持久化**：套装卡片 / 表格行出现「+N 未装」徽标，悬停点名是哪几个。
- 点徽标（或右键「安装上游新增」）确认后一键装进 store，按默认工具挂载，装完自动归拢回所属源套装。
- 健康横幅新增「上游新增 N 个未装」入口：点击循环定位到对应套装，旁边「全部安装」一次清账。
- 内容没变、只有上游新增时，检查结论如实说「内容已是最新；上游新增 N 个未装技能」——不再假装什么都没发生。

## `popskill://` 深链接

- `popskill://install?src=github.com/owner/repo` 从浏览器 / 文档 / 聊天一键唤起 Popskill 并直达安装计划页（预填地址、自动解析、照常人工确认后才安装）。
- 冷启动与运行中都接得住（AppDelegate + onOpenURL 双入口）；无参数时打开空白添加面板。

## 回收站恢复不再断更新链

移除条目时把来源记录（sourceUrl / 自动更新开关）随目录快照进回收站，恢复时自动回填——此前恢复回来的技能会变成「无来源」，从此检不了更新。回填不覆盖新记录，并清掉过期检查点防止误亮徽标。

## 未托管收编扩到全部四类

「导入未托管目录」此前只扫 skills/——现在 agents / mcp / bin 里手放的真实目录（含 Claude 单文件 agent .md）也会被发现并按类收编。

## 其它

- 环境自检：缺 git（装不了 GitHub 技能）或缺 npm（CLI 巡检不可用）时顶部横幅提示一次，可关闭不再提醒；探测全程后台，不阻塞启动。
- store 根目录被整个删掉再重建时，实时监听自动降级到父目录等待、重建后复活（此前会永久失联到下次切前台）。
- 测试 116 → 126；深链接经 LaunchServices 实测（打包 app 冷启 → `open popskill://…` → 日志确认进入添加流程）。
