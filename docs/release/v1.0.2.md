# Popskill v1.0.2 — UX fixes + landing-page polish

Quality-of-life release. No new features, but several user-visible bugs
that shipped in v1.0.0 / v1.0.1 are now fixed. v1.0.1 users will be
prompted by Sparkle automatically.

## What changed

### Bug fixes

- **Settings → "Reveal in Finder" no longer jumps to a non-existent
  directory.** The `Skills root (SSOT)` field displayed `~/.agents/skills`
  but sidecar actually stores SSOT at `~/.cc-switch/skills`. Both Settings
  and the Onboarding wizard now show the real path.
- **Matrix empty state no longer hides agent-only installs.** If you had
  agents installed but no skills, the matrix would show the "no
  capabilities yet" empty state even though the agents below were ready to
  render. Empty check now considers the full unified capability list.
- **Sidecar errors are now actually visible.** A failed sources fetch,
  notarize submit, or update check used to silently set `errorMessage` with
  no view ever reading it. A new global toast appears at the top of the
  window when something fails, with a dismiss button.

### Internal improvements (no user-visible change unless you look)

- **30-second refresh TTL** for Sources / Updates / Backups views. Toggling
  back and forth between sidebar entries no longer re-hits the sidecar
  every time. Manual refresh buttons still force a fresh fetch.
- **"Update all" button** is now properly disabled while a scan or per-row
  update is in flight, so rapid clicks can't stack concurrent updates.

## Migration

v1.0.1 → v1.0.2: Sparkle handles it. Open Popskill, accept the upgrade
prompt, restart. No data changes.

v1.0.0 → v1.0.2: v1.0.0 shipped without Sparkle config. Download v1.0.2
manually from this page; future versions will use Sparkle.
