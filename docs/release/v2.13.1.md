# v2.13.1 — 紧急修复：启动崩溃

修一个会让 v2.12 / v2.13.0 在**别人电脑上一打开就崩溃**的严重问题。

## 发生了什么

从 v2.12 引入界面双语起，app 用 Swift 包管理器生成的 `Bundle.module` 去找本地化资源。这个访问器写死了两个查找位置：app 包顶层、以及**打包那台机器上的一个绝对构建路径**。而资源实际放在 macOS 标准的 `Contents/Resources/` 里——两边对不上时它会直接 `fatalError` 崩溃。

在开发机上一直没暴露：那个绝对构建路径恰好存在，崩溃被掩盖了。只有在**没有这个路径的机器（也就是每一个真实用户）**上，一启动就崩。

## 修复

不再依赖 `Bundle.module`，改为自己在多个候选位置（首选 `Contents/Resources/`）查找资源包；**万一都找不到也绝不崩溃**——退回主包（界面回落中文），半成品也好过打不开。

打包冷启动烟测同步加固：测试前把开发机上的构建产物资源包移开，逼打包好的 app 只能用自己 `Contents/Resources/` 里的副本，精确复现「干净机器」的启动环境，这类崩溃以后会在 CI 就被挡下。

如果你装了 v2.12 或 v2.13.0 打不开，升级到这一版即可。
