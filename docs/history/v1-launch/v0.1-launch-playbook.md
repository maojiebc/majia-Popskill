# Popskill v0.1 Launch Playbook

This playbook is for the first public pre-alpha release after the DMG is signed, notarized, and verified. It keeps launch work separate from the engineering release runbook.

## Positioning

Lead with the narrow, honest claim:

- Popskill is a Mac-native local manager for AI capabilities, starting with Claude Code Skills and Agents.
- Popskill complements chat clients such as Cherry Studio. Cherry Studio is for talking to AI; Popskill is for managing the local capability library those tools use.
- Popskill is not claiming the Package or Bundle idea started here. The differentiation is the full Mac desktop combination: Skills, Agents, CLI tools, MCP, Config, Usage Insights, Stub, and AgentShield.

Avoid:

- Attacking Electron or cross-platform tools.
- Claiming first-ever capability packages.
- Opening more community surfaces than the project can maintain.

## Channel Plan

For v0.1:

- GitHub Release.
- GitHub Discussions.
- Product Hunt.
- HelloGitHub submission.
- V2EX or a small Chinese technical post.

Defer:

- Discord until v0.2 or clear inbound demand.
- QQ group until there is enough Chinese community load to justify moderation.
- Telegram until there is a repeatable update cadence.
- Broad localization beyond English and Simplified Chinese.

## Timeline

T-7:

- Share the signed DMG with 5 to 10 trusted technical testers.
- Collect install, Gatekeeper, first-run, and screenshot feedback.

T-3:

- Freeze release notes.
- Prepare screenshots and a short demo recording.
- Confirm `docs/release-runbook.md` and `docs/v0.1-qa-checklist.md` are satisfied.

T-1:

- Prepare Product Hunt draft.
- Prepare GitHub Release draft with `Popskill.dmg`, `release-manifest.json`, and `appcast.xml`.
- Re-run `POPSKILL_REQUIRE_RELEASE_METADATA=true POPSKILL_REQUIRE_SPARKLE=true ./scripts/release-doctor.sh`.

T:

- Publish GitHub Release after the uploaded DMG passes Gatekeeper from a clean macOS user account.
- Publish Product Hunt.
- Open GitHub Discussions.
- Post a concise Chinese launch note.

T+3:

- Triage install issues and release v0.1.1 if needed.
- Keep scope to crash fixes, signing/appcast issues, and obvious copy or visual defects.

T+7:

- Submit to HelloGitHub and Trendshift.
- Open focused PRs to relevant awesome lists.

T+14:

- Consider a small Hacker News or Reddit post if early tester feedback is stable.

T+30:

- Review whether v0.2 Package work, Discord, or a QQ group is justified by actual usage.
